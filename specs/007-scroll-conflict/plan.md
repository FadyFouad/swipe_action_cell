# Implementation Plan: Scroll Conflict Resolution & Gesture Arena

**Branch**: `007-scroll-conflict` | **Date**: 2026-02-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-scroll-conflict/spec.md`

---

## Summary

Replace the stock `GestureDetector(onHorizontalDragStart: ...)` in `SwipeActionCellState.build`
with a `RawGestureDetector` wrapping a new package-internal `SwipeHorizontalRecognizer extends
HorizontalDragGestureRecognizer`. The custom recognizer rejects gestures that (a) start within
the platform edge zone (`respectEdgeGestures`) or (b) are more vertical than horizontal
(`horizontalThresholdRatio`). A `NotificationListener<ScrollStartNotification>` outer wrapper
closes any open panel when the user initiates a vertical scroll (`closeOnScroll`). Three new
fields are added to `SwipeGestureConfig`; no new widget parameters, no new public types, no
breaking changes.

## Phase 0: Research ✅

See [research.md](research.md). All unknowns resolved.

Key decisions:
- **D1**: Custom `SwipeHorizontalRecognizer extends HorizontalDragGestureRecognizer` via `RawGestureDetector` — the only approach that participates correctly in Flutter's gesture arena without degrading `PageView` or nested-list behaviour.
- **D2**: Override `addAllowedPointer` (edge-zone check) and `handleEvent` (H/V ratio check using `kTouchSlop`). Do NOT override `hasSufficientGlobalDistanceToConsider`.
- **D3**: `ScrollStartNotification.dragDetails != null` distinguishes user-initiated from programmatic scroll — exact match for FR-008.
- **D4**: `GestureRecognizerFactoryWithHandlers` with mutable fields on the recognizer — config values updated via the initializer closure on each rebuild (avoids recreating the recognizer object).
- **D5**: Edge zone = `20.0` logical pixels from the left edge (`_kEdgeZoneWidth` constant).
- **D6**: Three new fields (`horizontalThresholdRatio`, `closeOnScroll`, `respectEdgeGestures`) on `SwipeGestureConfig` — inherits the three-level theme cascade for free.
- **D7**: No new state machine states — auto-close reuses `executeClose()` → `animatingToClose`.
- **D8**: All new fields are `const`-compatible primitives — `const SwipeGestureConfig()` stays valid.

### Phase B — New `SwipeHorizontalRecognizer`

New package-internal file. Tested indirectly via Phase C widget tests.

#### B1: Create the recognizer

**File**: `lib/src/scroll/swipe_gesture_recognizer.dart`

Full implementation per contracts/gesture-arena.md:

- Extends `HorizontalDragGestureRecognizer`
- Mutable fields `thresholdRatio` and `respectEdgeGestures` (updated by widget on each rebuild via the initializer closure)
- `static const double _kEdgeZoneWidth = 20.0`
- Private accumulators `_cumulativeH`, `_cumulativeV`, `_directionDecided` (reset in `addAllowedPointer`)
- `addAllowedPointer`: early return if `respectEdgeGestures && event.position.dx < _kEdgeZoneWidth`
- `handleEvent`: accumulate dx/dy; once `absH + absV > kTouchSlop`, check ratio; call
  `resolve(GestureDisposition.rejected)` if vertical dominates; set `_directionDecided = true`
  and call `super.handleEvent(event)` if horizontal dominates

**Note**: This file MUST NOT appear in `lib/swipe_action_cell.dart` or any other barrel.

#### C2: Add `_buildGestureRecognizers` method

```dart
Map<Type, GestureRecognizerFactory> _buildGestureRecognizers(double width) {
  return {
    SwipeHorizontalRecognizer:
        GestureRecognizerFactoryWithHandlers<SwipeHorizontalRecognizer>(
      () => SwipeHorizontalRecognizer(debugOwner: this),
      (instance) {
        instance
          ..thresholdRatio = effectiveGestureConfig.horizontalThresholdRatio
          ..respectEdgeGestures = effectiveGestureConfig.respectEdgeGestures
          ..onStart = _handleDragStart
          ..onUpdate = (d) => _handleDragUpdate(d, width)
          ..onEnd = (d) => _handleDragEnd(d, width);
      },
    ),
  };
}
```

#### C3: Add `_handleScrollStart` method

```dart
bool _handleScrollStart(ScrollStartNotification notification) {
  if (effectiveGestureConfig.closeOnScroll &&
      notification.dragDetails != null &&
      _state == SwipeState.revealed) {
    executeClose();
  }
  return false; // always bubble — do not absorb the notification
}
```

#### C4: Replace `GestureDetector` with `RawGestureDetector` + wrap in `NotificationListener`

Locate the existing `LayoutBuilder` in `build()` (currently starts around line 681) and replace:

```dart
// BEFORE:
LayoutBuilder(
  builder: (context, constraints) {
    final width = constraints.maxWidth;
    _widgetWidth = width;
    return GestureDetector(
      behavior: HitTestBehavior.translucent,
      onHorizontalDragStart: _handleDragStart,
      onHorizontalDragUpdate: (details) => _handleDragUpdate(details, width),
      onHorizontalDragEnd: (details) => _handleDragEnd(details, width),
      child: _wrapWithClip(AnimatedBuilder(/* ... */)),
    );
  },
),

// AFTER:
NotificationListener<ScrollStartNotification>(
  onNotification: _handleScrollStart,
  child: LayoutBuilder(
    builder: (context, constraints) {
      final width = constraints.maxWidth;
      _widgetWidth = width;
      return RawGestureDetector(
        behavior: HitTestBehavior.translucent,
        gestures: _buildGestureRecognizers(width),
        child: _wrapWithClip(AnimatedBuilder(/* ... */)),
      );
    },
  ),
),
```

**Note**: The two inner `GestureDetector` widgets (body-tap interceptor and confirmation
overlay tap) are unchanged — they handle tap events, not horizontal drag.
