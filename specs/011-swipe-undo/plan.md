# Implementation Plan: Swipe Action Undo/Revert Support

**Branch**: `011-swipe-undo` | **Date**: 2026-03-01 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/011-swipe-undo/spec.md`

---

## Summary

Add a time-limited undo mechanism to `SwipeActionCell`. After each action completes (progressive
increment or intentional trigger), an opt-in undo window opens for a configurable duration
(default 5 s). During this window the user can revert via a built-in overlay bar (shrinking
progress bar + "Undo" button) or programmatically via `SwipeController.undo()`. After expiry the
action commits permanently and `onUndoExpired` fires. A new `SwipeUndoConfig?` parameter enables
the feature; `null` (default) disables it with zero overhead. The undo animation for `animateOut`
reversal uses a distinct underdamped spring (slight bounce), clearly differentiated from normal
close animations. Integration with the F8 accessibility layer is automatic.

## Key Implementation Notes

### Zero-Overhead Guarantee

When `undoConfig == null`:
- `_undoBarController` is never created
- `_undoTimer` is never allocated
- `_undoPending` is always `false`
- `build()` never instantiates `SwipeUndoOverlay`
- Existing gesture and animation paths are fully unmodified

### Timer vs `AnimationController` Separation

Two clocks, independent roles:
- `Timer _undoTimer` — real-time expiry clock; fires `_commitUndo()` after `duration`
- `AnimationController _undoBarController` — purely cosmetic progress bar; not started under
  `reduceMotion` (`MediaQuery.disableAnimations`). Their durations match but lifecycle is independent.

### Group Controller Compatibility

`SwipeControllerGroup` calls `close()` on sibling cells. Since undo pending is a boolean flag
parallel to `SwipeState` (not part of the state machine), `close()` does NOT cancel the undo
window — satisfying the spec constraint that group-close does not cancel undo.

### `animateOut` Reversal Detail

When undoing an `animateOut` action, `_controller.value` is at `-(_widgetWidth * 1.5)` and
state is `animatingOut`. `_triggerUndo()` calls `_controller.animateWith(SpringSimulation)`
targeting `0.0` using `SpringConfig.undoReveal`. The existing `_handleAnimationStatusChange`
listener handles transition to `idle` when the animation settles — no special casing needed.

### Undo Window + New Gesture Race

`_handleDragStart()` checks `_undoPending` first. If true:
1. `_commitUndo()` fires `onUndoExpired` and clears state
2. Drag handling continues normally
3. After new action completes, `_startUndoWindow()` opens the new window

## Risks and Mitigations

| Risk                                                                                   | Mitigation                                                                                                               |
|----------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------|
| `_undoTimer` fires after widget `dispose()`                                            | `dispose()` calls `_undoTimer?.cancel()` before `super.dispose()`                                                        |
| Two clocks drift                                                                       | Timer is authoritative for expiry; AnimationController is cosmetic only — no shared state                                |
| Consumer calls permanent side effect in `onActionTriggered` instead of `onUndoExpired` | Documented prominently in quickstart.md with the correct pattern                                                         |
| `_lastPostActionBehavior` stale across rapid actions                                   | Reset at start of `_startUndoWindow()`; set in `_applyPostActionBehavior()` immediately before `_startUndoWindow()` call |
| `SwipeUndoOverlay` causes layout shift                                                 | Positioned within existing `Stack` — no layout impact on child                                                           |
