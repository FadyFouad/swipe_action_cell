## Summary

Add left-swipe intentional (one-shot) action behavior to `SwipeActionCell`. Two mutually
exclusive modes: **auto-trigger** (swipe past threshold → callback fires immediately, cell
applies `postActionBehavior`) and **reveal** (swipe opens a panel of 1–3 tappable action
buttons). The cell never interferes with F003's right-swipe progressive action. New public
types: `LeftSwipeMode`, `PostActionBehavior`, `SwipeAction`, `IntentionalSwipeConfig`,
`SwipeActionPanel`. Updated: `SwipeActionCell` (new `leftSwipe` parameter),
`SwipeState` (new `animatingOut` value), barrel exports.

- **Destructive expand**: `int? _expandedIndex` in `SwipeActionPanel`; `AnimatedContainer` width
- **Confirmation state**: `bool _awaitingConfirmation`; reuses `revealed` state; background area tap confirms, body tap cancels
- **`onSwipeCancelled` guard**: `bool _isPostActionSnapBack` (parallel to F003's `_isPostIncrementSnapBack`)

## Implementation Notes

### State Machine Hook

The existing `_handleAnimationStatusChange` completion branch:
```dart
} else if (_state == SwipeState.animatingToOpen) {
  if (_lockedDirection == SwipeDirection.right && widget.rightSwipe != null) {
    // F003 — unchanged
  } else {
    _updateState(SwipeState.revealed);  // existing left/unconfigured
  }
}
```

Becomes:
```dart
} else if (_state == SwipeState.animatingToOpen) {
  if (_lockedDirection == SwipeDirection.right && widget.rightSwipe != null) {
    // F003 — unchanged
  } else if (_lockedDirection == SwipeDirection.left && widget.leftSwipe != null) {
    _handleIntentionalActionSettled();   // NEW F004 path
  } else {
    _updateState(SwipeState.revealed);   // unchanged: unconfigured left
  }
}
```

### `_handleIntentionalActionSettled()`

```dart
void _handleIntentionalActionSettled() {
  final config = widget.leftSwipe!;
  if (config.mode == LeftSwipeMode.reveal) {
    _updateState(SwipeState.revealed);
    config.onPanelOpened?.call();
  } else {
    // autoTrigger
    if (config.requireConfirmation && !_awaitingConfirmation) {
      _awaitingConfirmation = true;
      _updateState(SwipeState.revealed);  // hold for user to confirm
    } else {
      _applyIntentionalAction();
    }
  }
}
```

### `_applyIntentionalAction()` and `_applyPostActionBehavior()`

```dart
void _applyIntentionalAction() {
  final config = widget.leftSwipe!;
  _awaitingConfirmation = false;
  if (config.enableHaptic) HapticFeedback.mediumImpact();
  config.onActionTriggered?.call();
  _applyPostActionBehavior();
}

void _applyPostActionBehavior() {
  final config = widget.leftSwipe!;
  switch (config.postActionBehavior) {
    case PostActionBehavior.snapBack:
      _isPostActionSnapBack = true;
      _updateState(SwipeState.animatingToClose);
      _snapBack(_controller.value, 0.0);
    case PostActionBehavior.animateOut:
      _updateState(SwipeState.animatingOut);
      _animateOut();
    case PostActionBehavior.stay:
      _updateState(SwipeState.revealed);
  }
}

void _animateOut() {
  final spring = widget.animationConfig.completionSpring;
  final simulation = SpringSimulation(
    SpringDescription(mass: spring.mass, stiffness: spring.stiffness,
        damping: spring.damping),
    _controller.value,
    -(_widgetWidth * 1.5),
    0.0,
  );
  _controller.animateWith(simulation);
}
```

### `animatingToClose` Completion — Callback Guards

```dart
if (_state == SwipeState.animatingToClose) {
  final wasIntentionalLeft = _lockedDirection == SwipeDirection.left &&
      widget.leftSwipe?.mode == LeftSwipeMode.autoTrigger;
  final wasPanelClose = _lockedDirection == SwipeDirection.left &&
      widget.leftSwipe?.mode == LeftSwipeMode.reveal;
  final wasProgressiveRight = _lockedDirection == SwipeDirection.right &&
      widget.rightSwipe != null;
  _lockedDirection = SwipeDirection.none;
  _awaitingConfirmation = false;
  _destructiveExpandedIndex = null;
  _updateState(SwipeState.idle);
  if (wasProgressiveRight && !_isPostIncrementSnapBack) {
    widget.rightSwipe!.onSwipeCancelled?.call();
  }
  if (wasIntentionalLeft && !_isPostActionSnapBack) {
    widget.leftSwipe!.onSwipeCancelled?.call();
  }
  if (wasPanelClose) {
    widget.leftSwipe!.onPanelClosed?.call();
  }
  _isPostIncrementSnapBack = false;
  _isPostActionSnapBack = false;
}
```

### Panel Open Target Offset

```dart
double _leftMaxTranslation(double widgetWidth) {
  final config = widget.leftSwipe;
  if (config?.mode == LeftSwipeMode.reveal && config!.actions.isNotEmpty) {
    return config.actionPanelWidth ?? _computeAutoPanelWidth(widgetWidth);
  }
  return widget.animationConfig.maxTranslationLeft ?? widgetWidth * 0.6;
}

double _computeAutoPanelWidth(double widgetWidth) {
  final count = widget.leftSwipe!.actions.length.clamp(1, 3);
  return (count * 80.0).clamp(60.0, widgetWidth * 0.65);
}
```

### Cell Body Tap Interceptor (Revealed State)

The translated child is conditionally wrapped when `_state == revealed && leftSwipe != null`:
```dart
Widget _maybeWrapWithBodyTapInterceptor(Widget child) {
  if (_state != SwipeState.revealed || widget.leftSwipe == null) return child;
  return GestureDetector(
    behavior: HitTestBehavior.opaque,
    onTap: _handleBodyTapInRevealedState,
    child: child,
  );
}

void _handleBodyTapInRevealedState() {
  if (_awaitingConfirmation) {
    _awaitingConfirmation = false;
    _updateState(SwipeState.animatingToClose);
    _snapBack(_controller.value, 0.0);
  } else {
    // reveal mode OR stay mode — close
    setState(() => _destructiveExpandedIndex = null);
    _isPostActionSnapBack = false;
    _updateState(SwipeState.animatingToClose);
    _snapBack(_controller.value, 0.0);
  }
}
```

### Background-Area Confirmation Tap

When `_awaitingConfirmation`, overlay a transparent `GestureDetector` over the background:
```dart
if (_awaitingConfirmation)
  Positioned.fill(
    child: GestureDetector(
      behavior: HitTestBehavior.translucent,
      onTap: _applyIntentionalAction,
    ),
  ),
```
`translucent` ensures body-area taps still reach `_maybeWrapWithBodyTapInterceptor`.

### Reveal Panel in `build()` Stack

```dart
if (widget.leftSwipe?.mode == LeftSwipeMode.reveal &&
    _state == SwipeState.revealed &&
    widget.leftSwipe!.actions.isNotEmpty)
  Positioned(
    top: 0, bottom: 0, right: 0,
    width: _computeEffectivePanelWidth(width),
    child: SwipeActionPanel(
      actions: widget.leftSwipe!.actions.take(3).toList(),
      panelWidth: _computeEffectivePanelWidth(width),
      onClose: () {
        setState(() => _destructiveExpandedIndex = null);
        _updateState(SwipeState.animatingToClose);
        _snapBack(_controller.value, 0.0);
      },
    ),
  ),
```

`_destructiveExpandedIndex` tracking is kept inside `SwipeActionPanel` itself (its own
`StatefulWidget` state), so the cell widget doesn't need to own it. The cell only needs
to reset when the panel closes (which `onClose` handles).

### Haptic for Left Swipe Threshold

```dart
// Inside AnimatedBuilder builder:
if (widget.leftSwipe?.enableHaptic == true &&
    _lockedDirection == SwipeDirection.left &&
    progress.isActivated &&
    !_hapticThresholdFired) {
  HapticFeedback.lightImpact();
  _hapticThresholdFired = true;
}
```

The existing `_hapticThresholdFired` resets in `_handleDragStart`, shared between
both directions (only one is active per gesture).

### Performance Considerations

- `SwipeActionPanel` is `StatefulWidget`; it rebuilds only when `_expandedIndex` changes
  (internally via `setState`). The main `AnimatedBuilder` does not trigger panel rebuilds.
- `AnimatedContainer` for destructive expand uses Flutter's implicit animation system —
  no `AnimationController` allocation, no frame-level callback on the main widget.
- The reveal panel Stack entry is `Positioned` (fixed geometry) — offset changes from the
  `AnimatedBuilder` do not cause panel relayout.
- `animateOut` drives the existing `_controller`; zero extra allocation.
- `_widgetWidth` is a simple field write in `LayoutBuilder` — no extra rebuilds.

### Test Strategy

Tests follow RED-GREEN pattern. All test tasks precede their implementation tasks.

**Unit tests** (`test/actions/intentional/`):
- `LeftSwipeMode` / `PostActionBehavior`: enum values, toString
- `SwipeAction`: equality, `copyWith`, `flex >= 0` assert, nullable `label`
- `IntentionalSwipeConfig`: equality, `copyWith`, `actionPanelWidth > 0` assert
- `SwipeActionPanel`: renders 1/2/3 buttons; destructive first tap expands; second fires
  `onTap` + calls `onClose`; non-destructive button fires immediately + calls `onClose`

**Widget tests** (`test/widget/swipe_action_cell_intentional_test.dart`):
- Auto-trigger: `onActionTriggered` fires on above-threshold release; nothing below threshold
- Fling left → triggers action
- `snapBack`: controller returns to 0.0; `onSwipeCancelled` does NOT fire
- `animateOut`: state → `animatingOut`; `onActionTriggered` fires; drag during animatingOut ignored
- `stay`: state → `revealed`; right swipe → back to idle
- `requireConfirmation`: first swipe → `revealed` (no callback); background tap → fires callback; body tap → snaps back
- Reveal panel opens: `onPanelOpened` fires; panel visible
- Reveal button tap: `onTap` fires; `onPanelClosed` fires; panel closes
- Reveal body tap: `onPanelClosed` fires; no `onTap`; panel closes
- Reveal right swipe: `onPanelClosed` fires; panel closes
- Reveal destructive expand: first tap expands; second tap fires `onTap` + `onPanelClosed`
- Haptic: light at threshold, medium on action fire (mocked platform channel)
- Coexistence (rightSwipe + leftSwipe): each direction fires only its own callbacks
- `leftSwipe: null`: zero regression against F001/F002/F003 test suite
