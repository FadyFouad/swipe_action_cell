## Summary

Add a visual background layer to `SwipeActionCell` that renders behind the sliding child during
swipe interactions. Developers supply `leftBackground` and `rightBackground` builder functions
that receive `SwipeProgress` on every animation frame and return a widget. A built-in
`SwipeActionBackground` widget provides a polished default (icon + optional label, scale/fade
animations, color intensification, threshold bump) for zero-boilerplate use. Clipping is
configurable via `clipBehavior` and `borderRadius`.

**New public surface**: `SwipeActionBackground` widget, 4 new `SwipeActionCell` parameters.
**Modified**: `SwipeActionCell` build method (Stack + clipping); progress computed unconditionally.
**Unchanged**: All F001 gesture, physics, and state-machine logic.

### Step 3 — Implement `SwipeActionBackground` (GREEN)

**File**: `lib/src/visual/swipe_action_background.dart`

Key implementation details:

- `_bumpController`: `AnimationController(duration: 300ms)`
- `_bumpAnimation`: `TweenSequence` — Item1: 0.0→0.3 (easeOut, weight 50), Item2: 0.3→0.0 (easeIn, weight 50)
- `_wasActivated`: tracks previous `isActivated` for transition detection
- `didUpdateWidget`: `if (isActivated && !_wasActivated) → _bumpController.forward(from: 0.0)`
- `build`: `AnimatedBuilder(animation: _bumpController)` wrapping:
  - `ColoredBox(color: _intensifiedColor(ratio))`
  - `Center → Opacity(opacity: ratio) → Transform.scale(scale: ratio * (1 + bump))`
  - `Column(mainAxisSize: min)` with `IconTheme(icon)` and optional `Text(label)`
- `_intensifiedColor`: HSL lightness − (0.15 × ratio), clamped to [0.0, 1.0]

### Step 6 — Implement `SwipeActionCell` background layer (GREEN)

**File**: `lib/src/widget/swipe_action_cell.dart`

**6a.** Add 4 new named parameters to constructor + fields (with dartdoc):
```
leftBackground: SwipeBackgroundBuilder?
rightBackground: SwipeBackgroundBuilder?
clipBehavior: Clip = Clip.hardEdge
borderRadius: BorderRadius?
```

**6b.** Add `_buildBackground(BuildContext, SwipeProgress) → Widget` to `_SwipeActionCellState`:
- Returns `SizedBox.shrink()` when `progress.direction == SwipeDirection.none`
- Returns `SizedBox.shrink()` when the direction's builder is null
- Otherwise calls the builder and returns the result

**6c.** Add `_wrapWithClip(Widget) → Widget` to `_SwipeActionCellState`:
- Returns `ClipRRect(borderRadius: r, clipBehavior: c, child: ...)` when `borderRadius != null`
- Returns `ClipRect(clipBehavior: c, child: ...)` when `clipBehavior != Clip.none`
- Returns child unchanged when `clipBehavior == Clip.none` and `borderRadius == null`

**6d.** Refactor `AnimatedBuilder.builder`:
- Move `SwipeProgress` computation outside `if (onProgressChanged != null)` — always compute
- Add fast path: if no builders configured, return `Transform.translate` directly (no Stack)
- Otherwise: return `Stack([Positioned.fill(background), Transform.translate(child)])`

**6e.** Wrap `AnimatedBuilder` in `_wrapWithClip(...)` inside the `GestureDetector.child`.

## Key Design Decisions

| Decision | Rationale | Reference |
|----------|-----------|-----------|
| Builder inside `AnimatedBuilder` (not `ValueNotifier`) | Avoids setState-during-build; no extra state fields | research.md Q1 |
| Fast path when no backgrounds configured | Avoids Stack overhead for cells without backgrounds | Step 6d |
| `SizedBox.shrink()` at idle (`direction == none`) | F001 state machine sets direction=none on snap-back completion naturally | research.md Q3 |
| `ClipRRect` / `ClipRect` two-branch strategy | Minimal render objects; standard Flutter pattern | research.md Q2 |
| `_bumpController` in `_SwipeActionBackgroundState` | Bump is an internal visual concern; `didUpdateWidget` detects transition cleanly | research.md Q4 |
| HSL lightness reduction for color intensification | Preserves hue; avoids desaturation artifact of lerp-to-black | research.md Q5 |
