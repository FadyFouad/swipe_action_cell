# Implementation Plan: Consolidated Configuration API & Theme Support

**Branch**: `005-config-api` | **Date**: 2026-02-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-config-api/spec.md`

**Note**: This is a breaking API-surface refactor of F001–F004. No observable widget behavior
changes. All new logic is type renaming, config consolidation, preset construction, and theme
inheritance.

---

## Summary

Rename `ProgressiveSwipeConfig` → `RightSwipeConfig` and `IntentionalSwipeConfig` →
`LeftSwipeConfig`, introduce `SwipeVisualConfig` to consolidate four top-level visual parameters,
add `SwipeActionCellTheme` as a `ThemeExtension`, add `tight()`/`loose()` gesture presets and
`snappy()`/`smooth()` animation presets, add an `activationThreshold` assertion, add a reveal-
mode empty-actions assertion, update `SwipeActionCell`'s constructor signature to the new API,
and implement theme-aware config resolution in `build()`.

---

## Technical Context

**Language/Version**: Dart >=3.4.0 <4.0.0
**Primary Dependencies**: Flutter SDK only (zero external runtime deps — Constitution IV)
**Storage**: N/A (stateless config objects)
**Testing**: `flutter test` (widget tests + unit tests)
**Target Platform**: All Flutter-supported platforms (iOS, Android, web, macOS, Windows, Linux)
**Project Type**: Flutter package (reusable library)
**Performance Goals**: 60 fps swipe interactions on mid-range devices (Constitution X); config
resolution in `build()` is synchronous and allocation-free beyond one nullable lookup
**Constraints**: `flutter analyze` zero warnings; `dart format` clean; `public_member_api_docs`
enforced; no new runtime dependencies
**Scale/Scope**: Public API surface — every parameter rename is a breaking change for consumers

## Phase 0: Research ✅

See [research.md](research.md). All unknowns resolved.

Key decisions:
- **R1**: New config types live in `lib/src/config/`; supporting types stay in `actions/`
- **R2**: `SwipeActionCellTheme` uses `ThemeExtension`; lookup via `maybeOf()` static helper
- **R3**: Preset values — `loose().deadZone=4.0`, `tight().deadZone=24.0` (6×); `smooth().completionSpring.stiffness=180.0`, `snappy().completionSpring.stiffness=700.0` (≈3.9×)
- **R4**: `SwipeVisualConfig` reuses existing `SwipeBackgroundBuilder` typedef
- **R5**: `SwipeController` extends `ChangeNotifier`; no-op body
- **R6**: Hard removal of old types; CHANGELOG migration section; minor version bump
- **R7**: `gestureConfig: null` falls through to theme → package default (not disabled)
- **R8**: `lerp()` hard cutover at `t >= 1.0`

### Phase C — `SwipeActionCellTheme`

#### C1: Theme type

**File**: `lib/src/config/swipe_action_cell_theme.dart`

```dart
class SwipeActionCellTheme extends ThemeExtension<SwipeActionCellTheme> {
  const SwipeActionCellTheme({
    this.rightSwipeConfig,
    this.leftSwipeConfig,
    this.gestureConfig,
    this.animationConfig,
    this.visualConfig,
  });

  final RightSwipeConfig? rightSwipeConfig;
  final LeftSwipeConfig? leftSwipeConfig;
  final SwipeGestureConfig? gestureConfig;
  final SwipeAnimationConfig? animationConfig;
  final SwipeVisualConfig? visualConfig;

  static SwipeActionCellTheme? maybeOf(BuildContext context) =>
      Theme.of(context).extension<SwipeActionCellTheme>();

  @override
  SwipeActionCellTheme copyWith({...}) => SwipeActionCellTheme(
        rightSwipeConfig: rightSwipeConfig ?? this.rightSwipeConfig,
        // ...
      );

  @override
  SwipeActionCellTheme lerp(
    ThemeExtension<SwipeActionCellTheme>? other,
    double t,
  ) {
    if (t >= 1.0) return (other as SwipeActionCellTheme?) ?? this;
    return this;
  }
}
```

**Tests** (`test/config/swipe_action_cell_theme_test.dart`):
- `const` constructability with no args
- `maybeOf(context)` returns `null` when no theme is installed
- `lerp(other, 0.0)` returns `this`
- `lerp(other, 0.5)` returns `this`
- `lerp(other, 1.0)` returns `other`
- `lerp(null, 1.0)` returns `this` (null safety)
- `copyWith` with no args returns equal instance
