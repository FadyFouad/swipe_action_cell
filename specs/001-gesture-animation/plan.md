## Implementation Notes

### Architecture Decisions

**Single AnimationController, pixel-space value**

The `AnimationController` stores the raw pixel offset (e.g., `+80.0` = 80px right,
`-50.0` = 50px left). It is unbounded (`lowerBound: double.negativeInfinity`). During drag,
`controller.value = newOffset` sets the position directly (synchronous, no animation tick).
During animation, `controller.animateWith(SpringSimulation(...))` drives the value via the
spring physics. `AnimatedBuilder` rebuilds the `Transform.translate` on every value change.

This avoids coordinate-space conversions and keeps the animation system simple.

**Dead zone and direction lock sequence**

On `onHorizontalDragStart`: call `controller.stop()`, reset accumulated delta and direction.

On `onHorizontalDragUpdate`:
1. While `_accumulatedDx.abs() < deadZone`: accumulate `delta.dx`, do NOT move the cell.
2. When threshold exceeded: lock direction. Check `enabledDirections` — if locked direction
   is disabled, swallow the drag silently.
3. Once locked: apply `delta.dx` through `applyResistance()` → assign to `controller.value`.

**Rubber-band resistance**

Use the iOS logarithmic formula from research.md Q4. Apply only when `abs(rawOffset) > maxT`.
Default `resistanceFactor = 0.55`. Factor `0.0` = hard clamp at `maxT`.

**Spring selection at release**

```
velocity = DragEndDetails.primaryVelocity ?? 0.0
ratio = controller.value.abs() / effectiveMaxTranslation
isFling = velocity.abs() >= velocityThreshold AND velocity aligns with locked direction
shouldComplete = isFling OR ratio >= activationThreshold
```

If `shouldComplete`: `animateWith(completionSpring, from=controller.value, to=±maxT, v=velocity)`
Else: `animateWith(snapBackSpring, from=controller.value, to=0.0, v=velocity)`

**State machine transitions**

Use `setState()` to update `_swipeState` and call `onStateChanged` callback. Call
`onProgressChanged` callback via `AnimatedBuilder`'s animation listener (not `setState`).

**`maxTranslation` resolution**

If `maxTranslation*` is `null`, resolve from the widget's `RenderBox` size after first layout.
Use `LayoutBuilder` or `after(frames: 1)` post-layout hook. Store the resolved value in state.
If layout hasn't completed yet (unlikely for list items), use a conservative fallback of 200.0px.

**Test exposure**

`_SwipeActionCellState` exposes `swipeOffsetListenable` as a `ValueListenable<double>` for
test access via `GlobalKey<_SwipeActionCellState>`.

### Performance Considerations

- `AnimatedBuilder` wraps only the `Transform.translate` — the child subtree is hoisted as
  the `child` parameter and NOT rebuilt on every animation frame.
- `controller.value = x` (direct assignment) does NOT call `setState` — it notifies
  `AnimatedBuilder` directly via the `Listenable` mechanism. This is O(1) per frame.
- `onProgressChanged` is called from `AnimatedBuilder`'s builder function — it fires on
  every animated frame automatically.
- The `applyResistance` function must be pure (no allocation) — uses only arithmetic.
